# PLAN v2 — Rock'n'Roll Game

Версия: 2.1
Дата: 2026-05-24
Статус: ключевые решения зафиксированы, ждём референсов по стилю

## 0. Принятые решения

- **Движок**: Godot 4 (free, HTML5 + Windows экспорт из одной кодовой базы, хороший 2D).
- **Графика**: 2D со слоями параллакса (2.5D-эффект без бюджета на 3D).
- **Музыка**: пока CC0 / CC-BY (Freesound, FMA, OpenGameArt), позже — оригинальные треки.
- **Сеть / лидерборды**: отложено в v3. В v2 — локальные рекорды (`localStorage` в вебе, файл на Windows).
- **Целевые платформы (в порядке приоритета)**: Web (HTML5) → Windows → дальше посмотрим.
- **Язык**: GDScript (быстрая итерация), узкие места при необходимости — на C# или GDExtension.

## 1. Концепция

Аркадный ритм-экшен в стиле классического рок-н-ролла 50–70-х годов. Игрок управляет музыкантом, попадая в ритм треков, отбиваясь от толпы скучающих слушателей и поднимая «энергию сцены». Основной упор — на отзывчивое управление с геймпада и узнаваемый ретро-визуальный стиль (винил, неон, кожа, хром).

Ключевые столпы:
- **Ритм + экшен** — нажатия привязаны к битам трека, но есть свобода перемещения.
- **Стиль превыше всего** — визуал, шрифты, эффекты — всё должно «звучать» как пластинка.
- **Геймпад first-class** — управление с контроллера ощущается как родное, клавиатура — равноправная альтернатива.

## 2. Цели версии v2

По сравнению с v1 (исходный набросок), v2 фокусируется на двух направлениях:

1. **Геймпад** — полная поддержка Xbox / DualSense / generic HID, ремаппинг, vibration feedback, dead-zone tuning.
2. **Визуальный стиль** — пересмотр палитры, шейдеров, типографики, переход на единый art direction.

Остальные системы (геймплей-петля, прогрессия, аудио) остаются как в v1 и трогаются только если упираемся в них.

| -------------- | ---------------------------------- | ------------------------------------------------------ |

## 4. Геймпад — детальный план

### 4.1 Поддерживаемые контроллеры
- Xbox One / Series (XInput, Bluetooth)
- DualSense / DualShock 4 (включая адаптивные триггеры на PS5-совместимых хостах)
- Switch Pro Controller
- Generic HID gamepads (через SDL mapping DB)

### 4.2 Маппинг по умолчанию
| Действие              | Xbox             | PlayStation      |
| --------------------- | ---------------- | ---------------- |
| Удар по струне 1      | A                | Cross            |
| Удар по струне 2      | B                | Circle           |
| Удар по струне 3      | X                | Square           |
| Удар по струне 4      | Y                | Triangle         |
| Соло / Spin           | RT               | R2               |
| Перемещение           | Левый стик       | Левый стик       |
| Камера / целеуказание | Правый стик      | Правый стик      |
| Pause                 | Start            | Options          |

### 4.3 Тонкие моменты
- **Dead zone**: радиальная, настраиваемая в опциях (0–30%).
- **Latency budget**: < 1 кадр от опроса до реакции (на 60 FPS — 16 мс).
- **Rumble**: трёхуровневый — лёгкий на промахах, средний на попаданиях, тяжёлый на критах.
- **Hot-swap**: переключение контроллера во время игры без рестарта.
- **Glyph swap**: автоматическая подмена иконок A/B/X/Y ↔ Cross/Circle/Square/Triangle по типу подключенного контроллера.

### 4.4 Ремаппинг
- UI-экран привязки кнопок.
- Сохранение профилей в `~/.config/rocknroll-game/input/<gamepad-id>.json`.
- Защита от «конфликтов» (одна кнопка — два действия).

## 5. Визуальный стиль

### 5.1 Mood board / референсы

> ⚠️ Ждём от автора уточнения: какую игру он называет «rocknroll» (Rock 'n' Roll Racing? Guitar Hero? что-то другое?). Пока — общий вектор:

- Обложки винила Sun Records и Atlantic 60-х.
- Постеры концертов Fillmore (Bill Graham).
- Неоновые вывески Лас-Вегаса 60-х.
- Comic-book хальф-тоны Kirby / Steranko.

**Принцип**: брать референс как стартовую точку, но «делать красивее» — современная типографика, чище композиция, агрессивнее работа со светом.

### 5.2 Палитра
| Имя       | HEX       | Назначение                       |
| --------- | --------- | -------------------------------- |
| Vinyl     | `#0A0A0A` | фон, сцена ночью                 |
| Neon Pink | `#FF2D87` | акцент, hit-effect               |
| Neon Cyan | `#2DE1FC` | UI, ритм-линейка                 |
| Hot Red   | `#E63946` | критический урон, fail-state     |
| Chrome    | `#D9D9D9` | металлические элементы, шрифт H1 |
| Cream     | `#F4E9CD` | основной текст                   |

### 5.3 Типографика
- **Заголовки**: Bebas Neue / Anton (узкий гротеск с харáктером концертной афиши).
- **UI**: Inter (нейтральный, читаемый).
- **Lyrics / flavor**: Special Elite (печатная машинка).

### 5.4 Визуальные эффекты
- CRT-overlay (опционально, в настройках) — сканлайны + лёгкая хроматическая аберрация.
- Bloom только на неоновых акцентах, не на всём кадре.
- «Виниловые царапины» при переходах между сценами.
- Particles на попаданиях — нотные знаки + искры.

### 5.5 Анимация
- Все анимации с лёгким overshoot (12 frames, ease-out back).
- UI пульсирует в такт BPM текущего трека.

## 6. Архитектура проекта (укрупнённо, Godot 4)

```
rocknroll-game/
├── docs/                ← планы, design docs
├── project.godot
├── assets/
│   ├── sprites/
│   ├── audio/           ← треки + sfx (CC-источники)
│   └── fonts/
├── scenes/
│   ├── main_menu.tscn
│   ├── stage.tscn       ← игровая сцена
│   └── results.tscn
├── scripts/
│   ├── core/            ← gameloop, scene manager
│   ├── input/           ← gamepad mapping, rebinding
│   ├── render/          ← shaders, post-processing
│   ├── audio/           ← beat sync, mixer
│   └── ui/
├── shaders/             ← .gdshader: CRT, bloom, scratches
└── tests/               ← gut / unit tests
```

## 7. Контроль качества

- **Юнит-тесты**: input mapping, beat detection, scoring math.
- **Интеграционные тесты**: прохождение treck-fixture с записанным input log.
- **Manual QA-чеклист** перед каждым релизом — см. `docs/QA_CHECKLIST.md` (создаётся отдельно).
- **Целевая частота кадров**: 60 FPS стабильно на интегрированной графике 5-летней давности.
