# PLAN v2 — Rock 'n' Roll Racing Clone

Версия: 2.3
Дата: 2026-05-24
Статус: концепция, стек и скоуп зафиксированы, готовы к Вехе 1

> **Референс**: [Rock 'n' Roll Racing](https://ru.wikipedia.org/wiki/Rock_n%E2%80%99_Roll_Racing) (Blizzard / Interplay, 1993, SNES/Genesis).
> Изометрический боевой гоночный аркадник с рок-саундтреком и комментатором.
> Задача — сделать **современный клон**: ту же душу, но красивее, отзывчивее и в браузере.

## 0. Принятые решения

- **Жанр**: изометрический combat-racer (клон Rock 'n' Roll Racing).
- **Движок**: Godot 4 (4.3+) — HTML5 + Windows из одной кодовой базы.
- **Графика**: 2D-спрайты в изометрической проекции (как оригинал) + современный пост-процессинг.
- **Музыка**: на старте CC0 / CC-BY рок-инструменталы (Freesound, FMA, OpenGameArt). Оригинал использовал Paranoid, Born to be Wild, Bad to the Bone, Highway Star, Radar Love — мы лицензировать их **не будем**, но стилистически целимся туда же.
- **Сеть / лидерборды**: v3. В v2 — локальные рекорды.
- **Платформы (по приоритету)**: Web (HTML5) → Windows → дальше посмотрим.
- **Язык**: GDScript, узкие места — C# / GDExtension при необходимости.
- **Сплит-скрин 2 игрока (локальный)**: **в v2**. Узнаваемая фишка оригинала.
- **Комментатор**: в v2 — простая озвучка из CC-сэмплов + текстовые реплики на экране. Качество доработаем позже.
- **Тач-управление в браузере (мобильные)**: **в v2**. Виртуальные стики + кнопки оружия.
- **Вибрация (rumble) геймпада на десктопе**: **обязательна**. В браузере — если позволяет API.

## 1. Концепция

Аркадные гонки с видом сверху-сбоку (изометрия), оружием и музыкой в духе хард-рока 70–80-х. Игрок едет по трассе на одной из 4–6 машин, стреляет в противников ракетами и минами, прыгает с трамплинов, собирает деньги и улучшения. Между гонками — гараж с апгрейдами и магазин новых машин.

Ключевые столпы:
- **Драйв и читаемость** — гонка должна быть «понятна за секунду», как в оригинале 1993 года.
- **Стиль превыше всего** — палитра, шрифты, эффекты — всё «звучит» как рок-пластинка.
- **Геймпад first-class** — рулёжка стиком обязана быть кайфом; клавиатура — равноправная альтернатива.
- **Камер-комментатор** — голос-войсовер реагирует на события («Loar Larsen ate dust!»). В v2 — текстовые реплики, в v3 — озвучка.

## 2. Цели версии v2

По сравнению с пустым репозиторием (v1 не было — это первая итерация плана) v2 фокусируется на:

1. **Геймпад** — полная поддержка Xbox / DualSense / generic HID, ремаппинг, **vibration на десктопе обязательна** (в браузере — если позволяет API), dead-zone tuning, **работа в браузере через Web Gamepad API**.
2. **Визуальный стиль** — пересмотр палитры/шейдеров/типографики, единый art direction «модернизированный RnRR».
3. **Сплит-скрин 2 игрока (локально)** — горизонтальный или вертикальный split, один геймпад + клавиатура, или два геймпада.
4. **Тач-управление в браузере** — для мобильных: виртуальные стики и кнопки.
5. **Озвучка комментатора (базовая)** — короткие CC-сэмплы на ключевые события.
6. **Вертикальный срез одной гонки** — играбельный круг с соперниками, оружием и финишем.

## 4. Геймпад — детальный план

### 4.1 Поддерживаемые контроллеры
- Xbox One / Series (XInput, Bluetooth)
- DualSense / DualShock 4
- Switch Pro Controller
- Generic HID (через SDL mapping DB, в браузере — Gamepad API)

### 4.2 Маппинг по умолчанию (гонка)

| Действие              | Xbox        | PlayStation  | Клавиатура       |
| --------------------- | ----------- | ------------ | ---------------- |
| Газ                   | RT          | R2           | W / ↑            |
| Тормоз / задний ход   | LT          | L2           | S / ↓            |
| Поворот               | Левый стик X | Левый стик X | A,D / ←,→        |
| Выстрел (основное)    | A           | Cross        | Space            |
| Мина / спец-оружие    | B           | Circle       | Shift            |
| Камера / обзор        | Правый стик | Правый стик  | —                |
| Пауза                 | Start       | Options      | Esc              |
| Бустер (если есть)    | X           | Square       | E                |

### 4.3 Тонкие моменты
- **Dead zone**: радиальная, настраиваемая (0–30%).
- **Latency budget**: < 1 кадр от опроса до реакции (60 FPS ⇒ 16 мс).
- **Rumble (десктоп — обязательно)**: лёгкий — занос/скрип шин, средний — попадание по тебе, тяжёлый — врезка/взрыв. В браузере используем `GamepadHapticActuator` если доступен; на отсутствие — тихо деградируем.
- **Hot-swap**: переключение контроллера без рестарта.
- **Glyph swap**: иконки A/B/X/Y ↔ Cross/Circle/Square/Triangle по типу контроллера.
- **Браузер**: переподключение геймпада в Chrome/Firefox — корректная обработка `gamepadconnected` / `gamepaddisconnected`.

### 4.5 Тач-управление (мобильный браузер)
- Левый нижний угол — виртуальный стик руления.
- Правый нижний угол — кнопки: газ, тормоз, ракета, мина.
- Опционально: тилт-управление как альтернатива стику (в настройках).
- Авто-определение тач-устройства → показываем оверлей; на десктопе — скрыт.
- Размер кнопок настраивается; safe-area iOS учитывается.

### 4.6 Сплит-скрин (локальные 2 игрока, десктоп)
- Два контроллера → автоматически активируется split-режим в меню.
- Альтернатива: 1 геймпад + клавиатура.
- Ориентация split: горизонтальная по умолчанию (как в оригинале на SNES), переключение в опциях.
- Мини-карта общая, по центру экрана, между двумя viewports.
- Звук — общий микс; rumble независимый для каждого игрока.

### 4.4 Ремаппинг
- UI-экран привязки кнопок.
- Сохранение профилей: `user://input/<gamepad-id>.json` (desktop), `localStorage` (web).
- Защита от конфликтов (одна кнопка — два действия).

## 5. Визуальный стиль

### 5.1 Mood board / референсы

- **База**: скриншоты Rock 'n' Roll Racing (SNES/Genesis, 1993) — изометрия, яркие машины, тёмный асфальт, цветные неоновые элементы трассы.
- Обложки винила Sun Records и Atlantic 60-х.
- Постеры концертов Fillmore (Bill Graham).
- Comic-book хальф-тоны Kirby / Steranko (для UI и портретов пилотов).

**Принцип «делать красивее»**:
- Сохраняем читаемость и силуэтность оригинала.
- Добавляем современный rim-light на машинах, мягкие тени, частицы.
- Палитра насыщеннее, но не «мультяшнее».
- Уходим от 16-битных ограничений (256 цветов) — используем полные 24 бита и градиенты.
- UI — не пиксель-арт, а чистая векторная типографика поверх пиксель-сцены.

### 5.2 Палитра

| Имя        | HEX       | Назначение                       |
| ---------- | --------- | -------------------------------- |
| Asphalt    | `#1A1A22` | дорожное полотно                 |
| Vinyl      | `#0A0A0A` | фон UI, тени                     |
| Neon Pink  | `#FF2D87` | акценты, hit-effect, ракеты      |
| Neon Cyan  | `#2DE1FC` | UI, индикатор скорости           |
| Hot Red    | `#E63946` | критический урон, fail-state     |
| Hot Yellow | `#FFD23F` | деньги, бустер, искры            |
| Chrome     | `#D9D9D9` | хром, корпуса машин              |
| Cream      | `#F4E9CD` | основной текст                   |

### 5.3 Типографика
- **Заголовки**: Bebas Neue / Anton (узкий гротеск как на афише концерта).
- **UI / счёт / скорость**: Inter (читаемый).
- **Реплики комментатора**: Special Elite (печатная машинка).

### 5.4 Визуальные эффекты
- CRT-overlay — **опционально** в настройках (сканлайны + лёгкая хроматическая аберрация).
- Bloom только на неоновых акцентах (выстрелы, фары, неоны на трассе).
- «Виниловые царапины» при переходах между сценами.
- Частицы: искры на скрипах шин, дым из выхлопной, обломки на взрыве.
- Screen shake на врезках и взрывах (настраивается, можно выключить).

### 5.5 Анимация
- Машины: легкий tilt на поворотах (sprite rotation ±5°), bounce на трамплинах.
- UI: overshoot 12 frames, ease-out back.
- Деньги при подборе летят к счётчику по кривой Безье.

## 6. Геймплей (что играем)

### 6.1 Игровая петля
1. Меню → 1 игрок / 2 игрока (сплит) → выбор трассы → выбор машины.
2. Гонка: 3–5 кругов, 3 соперника + игрок (в сплит-режиме — 2 игрока + 2 AI).
3. Финиш: позиция (1–4), бонус за фраги, бонус за деньги.
4. Гараж: тратим деньги на апгрейды (шины, броня, двигатель, оружие).
5. Следующая трасса / новая планета.

### 6.2 HUD во время гонки
- **Мини-карта** в углу (как в оригинале — см. референс-скриншот): общий вид трассы + точки игроков и соперников.
- Счётчик кругов: `LAPS 1/3`.
- Текущая позиция: `1st / 4`.
- Полоска здоровья машины.
- Иконки оружия / нитро с количеством зарядов.
- В сплит-режиме — отдельный HUD для каждого игрока; мини-карта общая по центру.

### 6.3 Машины (стартовый набор)
| Имя           | Особенность                            |
| ------------- | -------------------------------------- |
| Marauder      | Стартовая, баланс                      |
| Air Blade     | Быстрая, хрупкая                       |
| Battle Trak   | Танк, медленный                        |
| (+ 2–3 позже) | Раскрываются по прогрессу              |

### 6.4 Оружие
- Front missile (фронтальная ракета)
- Mine (мина сзади)
- Nitro (бустер ускорения)
- Plasma rifle (lvl 2, прокачка)

### 6.5 Трассы (план на v2)
- 1 планета («Drakonis»), 3 трассы разной сложности.
- В v3 — ещё 3 планеты по 3 трассы.

### 6.6 Комментатор
- В v2 — короткие CC-сэмплы (3–7 фраз): старт гонки, обгон, попадание, взрыв, финиш.
- Поиск голоса: Freesound (тег `announcer voice`), запись синтезом (ElevenLabs / встроенный TTS) — выбор по итогам прототипа.
- Текстовая реплика в углу экрана дублирует голос (доступность).
- Архитектурно — отдельный `commentator.gd` с очередью реплик и приоритетами (важное событие перебивает менее важное).

## 8. Контроль качества

- **Юнит-тесты**: input mapping, scoring math, car physics edge cases.
- **Интеграционные**: прохождение фиксированной трассы записанным input log.
- **Целевой FPS**: 60 стабильно в браузере на интегрированной графике 5-летней давности.
- **Manual QA-чеклист** перед релизом — `docs/QA_CHECKLIST.md` (создаётся отдельно).

## 9. Дорожная карта

- Godot 4 проект, структура папок из §7
- CI: GitHub Actions → HTML5 артефакт + публикация на GitHub Pages

### 9.4 Веха 4 — Оружие и бой (2 недели)
- Front missile, mine, nitro
- 1 AI-соперник

### 9.5 Веха 5 — Гонка целиком (2 недели)
- Lap counter, чекпойнты, финиш
- 3 AI-соперника с разной агрессией
- Мини-карта в HUD
- Экран результатов с очками и деньгами

### 9.6 Веха 6 — Сплит-скрин + тач (2 недели)
- Локальный сплит-скрин 2 игрока на 2 геймпадах (или геймпад+клава)

### 9.7 Веха 7 — Визуальный стиль закреплён (2 недели)
- Шейдеры (bloom, CRT-overlay, scratches)
- Финальный пасс по палитре всех экранов

### 9.8 Веха 8 — Вертикальный срез (2 недели)
- Меню → 1/2 игрока → выбор трассы → гонка → результаты → гараж → следующая гонка
